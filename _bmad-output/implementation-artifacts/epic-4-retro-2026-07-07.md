## 2. What Went Well

- **The E2E harness (4.5) justified itself on arrival.** Built to close the visual-verification gap, it *immediately found a real, shipped 4.3 defect*: the search input's `:focus` border never turned amber, because an inline `border` shorthand on `.kh-search-input` out-ranks the `.kh-search-input:focus{border-color}` rule (probe confirmed: focused, base border color, amber glow applied but border color did not). The fix was to the **product** (move the base border into the CSS class so `:focus` overrides color), not to the assertion. This is the exact class of defect the story exists to catch. **Borja confirmed this as a top-3 win.**
- **The cross-story intelligence loop kept closing prior-epic debt.** 4.1 resolved Epic 3 retro Action Item #2 (`deleted_at` exclusion semantics → EXCLUDE-IF-ANY anti-join, decision D1). 4.3 resolved Action Item #3 (view-in-Discord link → anchor `message_ids[0]` + `guildId` on `/api/auth/me`). Both closed at story creation, no re-work. **Borja confirmed as a win.**
- **Adversarial code review caught real correctness bugs, not noise.** 4.1: the all-zero query vector → NaN cosine similarity = garbage results, and non-finite → malformed `::vector` 500 (the guard Story 3.0's width-only assert missed). 4.2: an unbounded `page` param whose OFFSET would overflow int8 → 500 (capped at 1,000,000 with regression tests). **Borja confirmed as a win.**
- **AD-12 RBAC-inside-query made concrete for the read path.** Search and documents both filter by `allowedChannelIds` *inside* the query (`inArray`), never as a post-filter — proven by tests where a denied fragment identical to the query never surfaces. Run-unique test isolation (suffix-unique roles/channels + own-id cleanup) was applied where the full RBAC scope is asserted.
- **Epic 3 Action Item #1 was applied.** 4.2's round-2 review was a fresh independent pass over the round-1 patches — "treat every patch as new, un-reviewed code" is now practice.

## 3. Challenges

- **A story shipped with unverified visual ACs — the same "config-vs-reality surfaces late" pattern as Epic 3.** Story 4.3 was marked `done` with its CSS/font/box-shadow ACs never executed, because "gate green" (lint + unit + build) does **not** cover visual ACs: jsdom ignores external CSS, so visual criteria passed as un-verified disguised as green. The fix was Story 4.5 (a whole extra story), which verified 4.3 + 4.4 retroactively — and found the real `:focus` defect. **Insight (team + Borja): the lesson is not "more review"; it is that a visual AC is not "done" until the harness asserts it via `getComputedStyle`/screenshot.**
- **Playwright was assumed installed and was not.** The story-creation note "already installed" was false — Playwright was in no `package.json`, no `node_modules`, no bin. Same shape as Epic 3's configured-4096-vs-migrated-1536 dimensions and Epic 2's ioredis: an environment/tooling assumption that surfaced late. **Borja flagged this as a real pain.**
- **Fragile test isolation.** 4.2 uncovered a pre-existing bug where a broad `discord_id LIKE 'itest-%'` cleanup raced every other `itest-*` suite (dormant until this story's `user_read_status` FK write made it a 500), plus an RBAC expansion that resolves against the *whole* `channel_permissions` table, leaking other suites' channels into scope for a shared literal role. **Borja flagged this as a real pain.**
- **Inline-vs-cascade CSS debt carried forward.** The 4.3 `:focus` fix addressed the search input, but the *same* inline-`border`-shorthand-beats-`:hover` bug still disables `.kh-result-card:hover` and `.kh-chip:hover` border-color (flagged during 4.5, not fixed — no AC asserted them). Epic 5 adds substantially more interactive hover UI. **Borja flagged this as a real pain.**

## 9. Key Takeaways

1. **A visual AC is not "done" until the harness asserts it.** "Gate green" (lint + unit + build) does not cover CSS/visual criteria — jsdom ignores external CSS. Story 4.3 proved this the hard way; the 4.5 harness turned "looks right" into a `getComputedStyle` assertion and found a real defect.
2. **Environment/tooling assumptions surface late unless verified up front** — Playwright-not-installed is the same shape as Epic 3's embedding-dimension mismatch and Epic 2's ioredis. Verify at story creation.
3. **The cross-story intelligence loop keeps paying off** — Epic 4 closed two Epic 3 action items at story creation with zero re-work; extend it to Epic 5's retrieve query (which reuses 4.1's RBAC + `deleted_at` semantics).
4. **Deferred frontend debt now has a trigger.** `prefers-reduced-motion` (open since Story 2.1) and the inline-vs-`:hover` cascade both hit Epic 5's animated, interactive chat UI — close them *before* 5.3/5.4, not during.
5. **The SSE spike is Epic 5's early-validation spike** — validate the new streaming transport end-to-end before building the 5.4 UI on top of it.
