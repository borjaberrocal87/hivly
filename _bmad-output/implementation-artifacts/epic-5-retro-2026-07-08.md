## 3. Challenges

- **🔴 The multi-`system` Anthropic 400 — the king of this retro, and it is not fully fixed.** All testing uses a **fake** chat model, which does not enforce Anthropic's "only one leading `system` message" rule. Three review rounds on 5.1, the integration tests, and even the `curl` "live exercise" all passed green — because the fake model let two `system` turns (`SYSTEM_PROMPT` + `buildRAGContext`) through. The defect only surfaced when **Borja chatted against real Anthropic**. **Verified during this retro (three explore agents + Amelia's own read):** the uncommitted fix (merging the two static `system` turns at `graph.ts:90-93`) is **incomplete** — when history exceeds 4000 tokens, `compress.ts:98` returns a `<conversation summary>` `system` message that `reasonNode` prepends inside `...prepared`, producing **two leading `system` turns again → the same 400 in the compression path (5.2 AC3)**. **No test asserts "exactly one `system` at index 0."**
- **This is the "reality surfaces late" pattern — now on the LLM provider contract.** Same shape as Epic 3's 4096-vs-1536 embedding dimensions and Epic 4's Playwright-not-installed. Epic 4 AI#2 ("verify environment/tooling assumptions at story creation") covered tooling; the Anthropic single-`system` contract was exactly that class of assumption and slipped through — and it is precisely the one action item from Epic 4 that was only *partially* satisfied.
- **A test that lies (assertion tautology).** Repeatedly, a review round found the prior round's regression test would pass even if the fix were reverted (5.1 round-3's abort test where the fake threw unconditionally; the initial 5.2 fallback test that would pass with `[]`). The fix each time was to make the test *discriminate* — but the pattern recurs and needs to be a standing rule.
- **Cross-story contract consistency.** The multi-`system` bug is fundamentally this: 5.1 merged *its* two `system` turns, 5.2 later injected a *third* `system` turn (the compression summary) into the same `preparedMessages` structure, and no one reviewed the *combined* result. Each story was locally correct; the composed contract was not.
- **Spec-vs-reality planning gap: the execution-trace panel.** 5.4's AC and UX-DR20 describe a `tool_call`/`observation` panel the backend was never scoped to produce. Correctly deferred, but it means 5.4's AC as written is only partially met, and the capability was an un-owned backlog item until this retro (resolved in §5).

## 9. Key Takeaways

1. **Fake-model green is necessary, not sufficient.** The provider contract (Anthropic: one leading `system`) is the "reality surfaces late" pattern applied to the LLM — validate against the real provider before calling a chat story done.
2. **Local correctness ≠ composed correctness.** 5.1 and 5.2 were each right in isolation; the combined `preparedMessages` shape was wrong. Review the composed contract when stories feed the same structure.
3. **A test must discriminate its fix.** Multiple review rounds burned on tautological assertions that passed with the fix reverted — make revert-and-fail a DoD gate.
4. **"Every patch is new code" (Epic 3 AI#1) keeps earning its cost** — it caught the abort-swallow and two focus-trap regressions this epic, defects that lived entirely inside review patches.
5. **The prep-gate discipline works.** Honoring Epic 4's four critical-path items before 5.3/5.4 meant the frontend chat was built on solid ground — the only defect came from the one gate item (provider contract) that wasn't on the list.
